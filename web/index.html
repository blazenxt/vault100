Charter. The recorded charter of the bureau

What Vault100 is

Vault100 is a free, open-source, zero-knowledge file encryption app — the Seal Bureau where documents are locked into .v100 vaults. It runs in three bodies cut from one cloth: this web counter (which also installs as an offline PWA), a command-line engine (python -m vault100), and a desktop application — all compatible with one another, all under the MIT license on GitHub.

How a sealing proceeds

  1. Bring documents to window 1 — single files or a whole folder.
  2. Whisper a combination, pick a key-turning notch (108 of them), and optionally demand a physical second key (.v100key keyfile) or the double-lock cipher cascade.
  3. The clerk seals everything on your own device and hands back a .v100 vault — open it anywhere with window 2, the CLI, or the desktop app.

The locks behind the stamp

  • XChaCha20-Poly1305 secretstream — extended-nonce authenticated encryption with per-chunk tags and a tamper-evident header.
  • Optional AES-256-GCM ⟶ XChaCha20 cascade — two independent ciphers, both must fall for a break.
  • Argon2id memory-hard key turning — 108 notches up to 1 GiB × 6 turns; the device benchmark (the annex timekeeper) tells you what your own machine can afford.
  • Optional gzip shrink-wrap inside the vault, whole-folder tar filings, instant passphrase change that never re-encrypts the body.

Zero knowledge, by design

The bureau keeps no accounts, no telemetry, no escrow, and no copies. Every byte is encrypted inside a Web Worker on your device; the server only serves the printed paperwork — it never sees files, combinations, or keys. The source is open, the container format is documented, and every page proves itself by decrypting a reference vault sealed by the desktop app.

Questions the counter hears often

Is Vault100 really free?

Yes — free and MIT-licensed open source, with no accounts, no subscriptions and no ads. You may run it, study it, modify it and self-host it forever.

Do my files ever leave my device?

Never. All cryptography runs locally in a dedicated Worker thread of your browser (or fully offline in the CLI/desktop app). Disconnect the network and the bureau keeps working.

What if I forget a vault's combination?

No one on earth can recover it — not even the bureau, which keeps no ledger of combinations, keys or escrow copies. Write passphrases down and store keyfiles like house keys, with a copy laid safely away.

Which algorithms protect my vaults?

XChaCha20-Poly1305 authenticated streaming encryption, an optional AES-256-GCM cascade (double-lock), Argon2id memory-hard key derivation with 108 graded notches, and optional 256-bit keyfiles as a physical second factor. Parameters are stamped in each vault's public header, so any Vault100 build can open them.

Can I use Vault100 offline or self-host it?

Yes. After the first visit the web counter works with the network cable severed (PWA). Self-hosting is a zero-dependency Node server or the provided Dockerfile; the CLI and desktop app never need a network at all.

Can I change a vault's passphrase later?

Yes — Form 100-R (or python -m vault100 passwd) re-seals only the vault's key wrap. The body is never re-encrypted, so a 10 GB vault's combination changes as fast as a 10 KB one.
